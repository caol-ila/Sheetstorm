## Key Patterns

- **Cursor Encoding:** `Base64(ID:timestamp)` — reversible, safe for URLs
- **Stable Ordering:** All queries use composite key (BandId, CreatedAt/StartsAt desc, Id)
- **Determinism:** Same cursor always produces same starting point (no race conditions)
- **Backward Compat:** Queries without cursor return ALL results (original behavior)
- **Next-Page Logic:** If result set < pageSize, `hasMore=false` and `nextCursor=null`
