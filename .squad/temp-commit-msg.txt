ARCHITECTURE & DECISIONS

1. Annotations-Sync
   - Single AnnotationController for band-scoped and personal endpoints
   - Soft-delete (IsDeleted flag) for sync consistency
   - Version-based concurrency with 409 Conflict response
   - Orchestra annotations require Conductor/Admin create role
   - Decisions documented in .squad/decisions.md

2. Metronom
   - BeatCalculator: pure math (no Riverpod), 21 unit tests
   - ClockSyncService: instance variable in MetronomeNotifier (not global)
   - Sentinel-pattern for nullable copyWith fields (MS2 bug fix)
   - Route: /app/metronome?conductor=true (local role check, no server roundtrip)
   - Decisions documented in .squad/decisions.md

3. Frontend Patterns
   - Manual SignalR JSON protocol (same as BroadcastSignalRService)
   - Offline queue in Riverpod state with atomic dequeue
   - LWW (Last-Write-Wins) conflict resolution: timestamp > version tiebreaker
   - REST endpoints: /api/bands/{bandId}/* (no version segment, camelCase JSON)

4. Cloud-Sync
   - REST only (no WebSocket), multipart/form-data upload
   - tar.gz bundles for download
   - User-driven conflict resolution (no auto-merge)
   - Status indicator invisible on success
