# Setup Playwright E2E test infrastructure

**Labels:** `milestone:1.5`, `testing`, `e2e`, `infrastructure`

## Beschreibung

Einrichtung der kompletten Playwright E2E-Test-Infrastruktur für die Flutter-Web-Anwendung Sheetstorm.

## Aufgaben

- [ ] Playwright installieren (`npm init playwright@latest`)
- [ ] Konfiguration für Flutter Web erstellen (`playwright.config.ts`)
  - Base URL auf Flutter-Web-Dev-Server konfigurieren
  - Browser: Chromium, Firefox, WebKit
  - Timeouts für Flutter-Rendering anpassen (längere Wartezeiten wegen Dart-Kompilierung)
- [ ] Verzeichnisstruktur `sheetstorm_app/e2e/` anlegen:
  ```
  sheetstorm_app/e2e/
  ├── playwright.config.ts
  ├── package.json
  ├── tests/
  │   ├── auth/
  │   ├── import/
  │   ├── player/
  │   └── settings/
  ├── fixtures/
  │   └── base.fixture.ts    # Custom fixtures (auth state, test data)
  ├── helpers/
  │   └── flutter-utils.ts   # Flutter-spezifische Selektoren & Waits
  └── .env.example
  ```
- [ ] npm-Scripts hinzufügen:
  - `npm run e2e` — Alle E2E-Tests ausführen
  - `npm run e2e:headed` — Tests mit sichtbarem Browser
  - `npm run e2e:debug` — Tests im Debug-Modus
  - `npm run e2e:report` — HTML-Report generieren
- [ ] Flutter-Web-spezifische Helper erstellen:
  - Warten auf Flutter Engine-Initialisierung
  - Semantics-Tree-basierte Selektoren
  - Screenshot-Vergleich Setup
- [ ] CI-Integration vorbereiten (GitHub Actions Workflow-Stub)
- [ ] `.github/copilot-instructions.md` ergänzen: E2E-Tests sind Pflicht für alle user-facing Workflows

## Akzeptanzkriterien

- [ ] `npm run e2e` führt mindestens einen Smoke-Test erfolgreich aus
- [ ] Playwright-Report wird generiert
- [ ] Verzeichnisstruktur steht und ist dokumentiert
- [ ] copilot-instructions.md enthält E2E-Regeln
