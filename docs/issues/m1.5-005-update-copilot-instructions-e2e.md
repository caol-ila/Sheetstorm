# Update copilot-instructions.md: E2E test requirements

**Labels:** `milestone:1.5`, `testing`, `e2e`, `documentation`

## Beschreibung

Die `.github/copilot-instructions.md` um E2E-Test-Anforderungen und Playwright-Konventionen erweitern.

## Aufgaben

### Neue Regel hinzufügen
- [ ] Regel: **Alle user-facing Workflows MÜSSEN E2E-Coverage haben**
  - Jedes Feature mit UI-Interaktion benötigt mindestens einen E2E-Test
  - E2E-Tests sind Merge-Blocker (CI muss grün sein)
  - Neue Features ohne E2E-Tests werden im Review abgelehnt

### Playwright-Konventionen dokumentieren
- [ ] Dateistruktur:
  ```
  sheetstorm_app/e2e/tests/{feature}/*.spec.ts
  ```
- [ ] Naming: `{feature}.{scenario}.spec.ts` (z.B. `auth.login-success.spec.ts`)
- [ ] Test-Organisation: `test.describe()` pro Feature, `test()` pro Szenario
- [ ] Page Object Pattern für wiederverwendbare Selektoren
- [ ] Fixtures für Auth-State und Test-Daten

### Anleitung: E2E-Tests ausführen
- [ ] Dokumentieren:
  ```bash
  cd sheetstorm_app/e2e
  npm run e2e           # Alle Tests headless
  npm run e2e:headed    # Mit Browser-Fenster
  npm run e2e:debug     # Playwright Inspector
  npm run e2e:report    # HTML-Report öffnen
  ```
- [ ] Voraussetzungen: Flutter Web Dev-Server muss laufen
- [ ] Environment-Variablen für Test-Credentials

## Akzeptanzkriterien

- [ ] copilot-instructions.md enthält E2E-Pflicht-Regel
- [ ] Playwright-Konventionen sind klar dokumentiert
- [ ] Ausführungsanleitung ist vollständig und korrekt
