# E2E: File Import flow (Upload → Labeling → Metadata)

**Labels:** `milestone:1.5`, `testing`, `e2e`

## Beschreibung

End-to-End-Tests für den Noten-Import-Workflow: Datei hochladen, Seiten extrahieren, Labeling durchführen und Metadaten bearbeiten.

## Testfälle

### Datei-Upload
- [ ] PDF-Datei über File-Picker hochladen
- [ ] Bild-Datei (PNG/JPG) hochladen
- [ ] Mehrere Dateien gleichzeitig hochladen
- [ ] Upload-Fortschrittsanzeige wird korrekt dargestellt
- [ ] Fehlermeldung bei ungültigem Dateityp
- [ ] Fehlermeldung bei zu großer Datei

### Seitenextraktion (Status-Polling)
- [ ] Nach Upload: Seitenextraktion startet automatisch
- [ ] Status-Polling zeigt Fortschritt an (Spinner/Progress)
- [ ] Extrahierte Seiten werden als Thumbnails angezeigt
- [ ] Timeout-Handling bei langer Verarbeitung

### Labeling (Seite → Song-Gruppierung)
- [ ] Seiten können einem Song zugeordnet werden
- [ ] Neue Songs können erstellt werden
- [ ] Drag & Drop oder Klick-Zuordnung funktioniert
- [ ] Seiten-Reihenfolge innerhalb eines Songs ändern
- [ ] Nicht zugeordnete Seiten werden hervorgehoben

### Metadaten-Editor
- [ ] Song-Titel bearbeiten
- [ ] Komponist/Arrangeur eintragen
- [ ] Stimme zuweisen (aus Dropdown)
- [ ] Metadaten speichern und Bestätigung erhalten
- [ ] AI-basierte Metadaten-Vorschläge (falls AI-Key konfiguriert)

## Technische Hinweise

- Test-PDF-Dateien in `e2e/fixtures/` bereitstellen
- File-Picker benötigt `setInputFiles()` von Playwright
- Status-Polling: `waitForResponse` oder Custom-Polling-Helper verwenden
- Backend-Mock oder echtes Backend mit Seed-Daten
