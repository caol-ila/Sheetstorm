# E2E: Auth flow (Login → Onboarding → Library)

**Labels:** `milestone:1.5`, `testing`, `e2e`

## Beschreibung

End-to-End-Tests für den kompletten Authentifizierungs- und Onboarding-Flow in Sheetstorm.

## Testfälle

### Login
- [ ] Erfolgreicher Login mit Demo-Account (E-Mail + Passwort)
- [ ] Fehlermeldung bei ungültigen Credentials (falsches Passwort)
- [ ] Fehlermeldung bei nicht-existierender E-Mail
- [ ] Formular-Validierung (leere Felder, ungültiges E-Mail-Format)
- [ ] Login-Button deaktiviert während Request läuft

### Onboarding
- [ ] Nach erstmaligem Login: Onboarding-Flow wird angezeigt
- [ ] Onboarding-Schritte durchlaufen (Instrument wählen, Kapelle beitreten/erstellen)
- [ ] Weiterleitung zur Bibliothek nach Abschluss
- [ ] Onboarding wird bei erneutem Login nicht erneut angezeigt

### Library-Redirect
- [ ] Nach Login landet User in der Noten-Bibliothek
- [ ] Navigationsleiste zeigt korrekten Benutzernamen
- [ ] Geschützte Routen sind nach Login erreichbar

### Logout
- [ ] Logout-Button funktioniert
- [ ] Nach Logout: Redirect zur Login-Seite
- [ ] Geschützte Routen nach Logout nicht mehr erreichbar
- [ ] Token wird aus Storage entfernt

## Technische Hinweise

- Demo-Account-Credentials als Environment-Variable in `.env.test`
- Auth-State als Playwright Storage State speichern für andere Tests
- Timeout-Handling für Flutter-Rendering beachten
