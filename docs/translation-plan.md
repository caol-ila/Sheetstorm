The Flutter Drift schema (`app_database.dart`) defines local tables. A schema migration version bump is required to rename tables/columns. Existing local data will need migration logic.

## 6. Risk Assessment

### High Risk

| Risk | Impact | Mitigation |
|------|--------|------------|
| **API contract mismatch** | Frontend calls wrong endpoints → app completely broken | Both sides use this document as single source of truth. Integration test in Phase 4. |
| **DB migration data loss** | Renaming tables/columns incorrectly drops data | Use `RenameTable`/`RenameColumn` in migration, never drop+recreate. Test migration on copy of data. |
| **Config key migration** | Existing config values become orphaned with old key names | Data migration script must update all stored key strings. |
| **Enum serialization mismatch** | JSON serialization of renamed enums breaks API contract | Explicitly verify JSON enum values match between C# and Dart. Consider `[JsonPropertyName]` attributes. |

### Medium Risk

| Risk | Impact | Mitigation |
|------|--------|------------|
| **Missed German identifier** | Inconsistency, partial translation | Thorough file scan (this document), grep verification after implementation. |
| **Generated file conflicts** | `.g.dart` files not regenerated, build fails | Run `build_runner` after all source changes. Delete and regenerate if needed. |
| **Merge conflicts** | Other branches have German names, conflict on merge | Freeze other work during translation. Communicate to all agents. |
| **Test assertion strings** | Tests check German error messages that are now English | Update all assertion strings in the same pass. |
| **Client-side Drift migration** | Local DB on user devices has German schema | Implement Drift schema migration with version bump. |

### Low Risk

| Risk | Impact | Mitigation |
|------|--------|------------|
| **File rename in Git** | Git may not track renames properly | Use `git mv` for file renames where possible. |
| **Namespace changes** | `using` statements break across project | IDE refactoring or grep-and-replace all `using` statements. |
| **Documentation references** | Docs reference German code names | Docs stay German per directive — but cross-references from docs to code may need updating. |
